On 5 February 2013 10:49, Edward d'Auvergne <edward@xxxxxxxxxxxxx> wrote:

Hi Stacy,

I have now fixed all bugs preventing the model-free analysis in the
GUI from running without a PDB file (this is in the
dauvergne_protocol.py analysis, see
http://www.nmr-relax.com/manual/new_protocol_in_GUI.html and
http://www.nmr-relax.com/manual/d_Auvergne_protocol_script_mode_sample_script.html).
 You may find other bugs as this 'code path' is not very well tested
in the relax test suite.  Hence when I perform something called 'code
refactoring' to improve relax, breakages may occur that I don't see.
The aim for the relax test suite is to cover all possible ways of
using relax, but unfortunately this coverage is incomplete.  If you do
see any more issues, could you please submit a bug report.  If you
attach files with the minimum amount of data to reproduce the bug, I
can then add this to the test suite.  For example, I hope you don't
mind but I added your data for residues 9 and 10 here
http://svn.gna.org/viewcvs/relax/trunk/test_suite/shared_data/model_free/bug_20479_gui_final_pipe/
and 
http://svn.gna.org/viewcvs/relax/trunk/test_suite/shared_data/saved_states/bug_20480.bz2?view=log.
 The addition of this data and new tests means that the problems you
are seeing can never come back to relax.

There are two ways you can obtain these fixes.  The first is the
simplest - you could wait until I release a new version of relax.
Though when a new release occurs is unknown (pressure from relax users
helps).  Alternatively you can use the subversion program to obtain a
copy of relax from the source code repository (for details see
http://www.nmr-relax.com/manual/Latest_sources_relax_repositories.html
or http://www.nmr-relax.com/download.html#Source_code_repository).

Regards,

Edward
