Posted by Edward d'Auvergne on February 25, 2010 - 01:20:
Hi,

The status information container is the way to go here.  See
https://mail.gna.org/public/relax-devel/2010-01/msg00094.html.  But
the idea is simple, one part of the code (user function) sets a
variable, and the other part (relax controller) reads it.  No need to
do any text parsing of what comes out of STDOUT.  The only problem
might be locking, but if this is an issue there should be something in
the threading documentation that can help.  With this type of setup,
you can have the user functions place any info into the container, and
the relax controller GUI element can do with it as it pleases.
Numbers, spin name, spin index, MC sim number, etc.

Regards,

Edward
