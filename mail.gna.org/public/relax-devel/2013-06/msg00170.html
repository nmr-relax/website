Hi Troels,

This file looks fine.  It would be good to have a commit message
explaining in detail what the file is and why it is being added to the
repository.  Just a quick reminder of the format:

- The first line should be a quick summary under 100 characters.  If
sufficient and not too trivial, I use this for the relax release
announcement.
- A blank line.
- A description and link to the support request.
- Another blank line.
- All the details in multiple paragraphs if needed.
- The final blank line.

Everything but the first line should be word wrapped to 100 characters.

Cheers,

Edward
