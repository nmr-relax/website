Posted by Edward d'Auvergne on May 07, 2013 - 14:07:
A few weeks ago, the relax API section of the website was modified.
This is mainly for allowing for the preservation of links in the
mailing lists and Gna! trackers, as the relax internal API is in a
state of constant flux.  Hence the hyperlinks for different relax
versions are often not preserved.  With the upcoming relax 3.0.0
release, the layout of the source code has changed so much that most
of the relax 2.2 API links would no longer be functional.  Therefore I
have created a main page (http://www.nmr-relax.com/api/) which then
points to API documentation for each minor relax version number:

relax 2.2.5 API documentation (http://www.nmr-relax.com/api/2.2/),
relax 2.1.2 API documentation (http://www.nmr-relax.com/api/2.1/),
relax 2.0.0 API documentation (http://www.nmr-relax.com/api/2.0/),
relax 1.3.16 API documentation (http://www.nmr-relax.com/api/1.3/),
relax 1.2.15 API documentation (http://www.nmr-relax.com/api/1.2/),
relax 1.0.9 API documentation (http://www.nmr-relax.com/api/1.0/).

Again for preserving links, the old HTML files located in the base
directory have been preserved, for example
http://www.nmr-relax.com/api/minfx-module.html, but have no direct
links from http://www.nmr-relax.com.

Regards,

Edward
