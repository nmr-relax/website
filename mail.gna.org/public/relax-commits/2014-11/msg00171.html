  Important bug fix for racing causing the GUI to freeze.
  
  This is really only seen in the GUI tests on MS Windows systems, as a user 
could never be fast
  enough with the mouse.  The GUI interpreter flush() method for ensuring 
that all user functions in
  the queue have been cleared now calls wx.Yield() to force all wxPython 
events to also be flushed.
  This change will avoid random freezing of the relax test suite.

structure.load_spins user
  function calls.  Without this, the test will freeze on MS Windows.  The 
freezing behaviour is
  however not 100% reproducible and is dependent on the Windows version and 
wxPython version.
