Hi,

Sorry, I fixed the bug but haven't sent the message as to how to
benefit from it.  To obtain the fixes you can do one of two things -
either wait for a new version of relax to come out (which can take
quite a while unless there is something urgent) or to check out the
code from the repository.  For the second option, see one of these
links for instructions:

https://mail.gna.org/public/relax-users/2009-02/msg00020.html
https://mail.gna.org/public/relax-users/2009-08/msg00001.html
http://www.google.com/search?q=%22svn+co%22&btnG=Google+Search&domains=http%3A%2F%2Fnmr-relax.com&sitesearch=https%3A%2F%2Fmail.gna.org%2Fpublic%2Frelax-users

(the last link is a google search of the relax-users mailing list from
http://www.nmr-relax.com/communication.html for the necessary
instructions).

The issue here is just a lack of data compared to the number of
parameters.  3 relaxation data points are used so models tm4-tm8 which
have 4 or more parameters cannot be optimised.  These are now
deselected prior to optimisation with a RelaxWarning message saying so
and relax executes normally performing optimisation of only models
tm0-tm3 and tm9 and finishes executing the script without error.  I
hope this is what you were after.

Regards,

Edward
