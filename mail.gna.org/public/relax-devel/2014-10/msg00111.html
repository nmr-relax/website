Follow-up Comment #2, bug #22818 (project relax):

The problem is a number of memory leaks.  Parts of the GUI are not being
properly destroyed.  This can be seen in the Windows Task Manager.  In the
processor tab, use 'View->Select columns' to add the 'Handles' and 'USER
Objects'.  Then run the test suite.  The failure occurs once the 'USER
Objects' count for the process reaches 10,000, the maximum limit in MS
Windows.

This is why the execution of the failing User_functions GUI tests by
themselves does not trigger the bug.
