Posted by Edward d'Auvergne on June 23, 2014 - 11:55:
Hi,

These really should all be numpy.complex128 - the double precision
complex floats.  Setting this to numpy.complex64 drops the accuracy to
single precision which may introduce truncation artefacts.  Note
though that if all data structures in a dispersion model can be
converted from double to single precision, that the speed of that
model should be significantly faster.

Regards,

Edward
