Hi,

This is again another strange problem.  It should be quite fixable
though.  In relax, the R1 and R2 curve-fitting goes through exactly
the same code.  They are not differentiated in any way, and the same
script can be used for both.  Therefore the issue is not one of R1 vs.
R2.  The problem would again likely be due to the input data, most
likely some partially missing points.  If you could try the same
procedure as before, this would be of significant help in working out
what is happening and solving this problem.

If you could submit a bug report (https://gna.org/projects/relax/),
that would be much appreciated.  If you can find the minimal set of
peak lists which triggers the bug - ideally one or two lines, together
with a script, attaching these to the bug report will allow me to find
and fix the bug within literally minutes.  Unfortunately the message
below is currently insufficient for me to dig out the bug.

Cheers,

Edward
