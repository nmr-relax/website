2013/7/18 Edward d'Auvergne <edward.dauvergne@xxxxxxxxx>:

Hi,

In preparation for the release of relax 3.0.0, there are a few
outstanding issues which need to be addressed.  Note that the
relaxation dispersion branch will not be merged back into the mainline
until after this release.  The dispersion code will form the backbone
of the relax 3.1.x releases in the near future.  The 3.0.0 release
will deal mainly with the huge infrastructure changes which have
occurred since relax 2.2.5.

The issues in the relax trunk are visible when running the relax test
suite.  There is one system test failing to do with the NMRPipe
seriesTab peak list format support:

$ relax -s Peak_lists.test_read_peak_list_NMRPipe_seriesTab

There is also a related unit test failure.  We need to address these
issues before a new version can be released.  Once the test suite
passes on all operating systems, then the jump to relax 3 can occur.

Cheers,

Edward
