Follow-up Comment #2, bug #21324 (project relax):

I am happy to report simple solution to this problem.

The problem is based on the wxPython version and 32/64 bit issues.

If one installs the "epd" enthought python distribution, this includes a
working wxPython.
https://www.enthought.com/repo/epd/installers/
Specifically download and install:

epd-7.3-2-macosx-i386.dmg

A specific note here:
Special note about 64-bit Mac OSX

-----------
The macosx-x86_64 installer does not include wxPython, which a lot of ETS
relies on. Even though ETS supports the use of PySide as a backend, things 
are
much less stable under Qt. Therefore, if you want to use ETS, we recommned
using the 32-bit EPD on Mac OSX.
---------

The test suite now runs fine, and there is no problem with the GUI.
