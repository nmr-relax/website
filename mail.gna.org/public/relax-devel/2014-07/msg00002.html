Follow-up Comment #1, bug #22257 (project relax):

This is a clear bug in relax, and it a recurring problem in Mac OS X as it
cannot be tested in the relax test suite (so if you find any other problems,
please report these as well).  The problem is with wxPython.  The dialogs when
modal, such as the free file format window, cannot be destroyed on Mac OS X
using wx.Dialog.Destroy() - this crashes and freezes wxPython and hence kills
relax.  The problem does not exist on any other operating system.
