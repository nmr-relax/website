Details:

For a better web presence of http://www.nmr-relax.com, one page should be
created for each analysis type supported by relax.  The aim is that if someone
searches on the web for 'model-free software', 'relaxation dispersion
software', etc, then the corresponding relax web page would appear at position
1 of their search results.  The title should be crystal clear with the
analysis name and 'software' in it.

These pages could be accessible from a new side menu entry 'Analyses'.  This
could be collapsible via CSS so when clicked on, a list of each analysis type
could be shown.  But the main analysis page would also open when clicked on
and have a list of all the pages as well.

Each analysis page should include all the keywords for that analysis type. 
And it could have a screenshot of the GUI for that analysis, if one exists. 
They could be mini versions of the chapters of the relax manual.  So a general
layout proposal is:

- Description
- Script UI
- GUI
- Wiki

The wiki section could include a link to a special category on
http://wiki.nmr-relax.com for that analysis type, for example:

http://wiki.nmr-relax.com/Category:NOE
http://wiki.nmr-relax.com/Category:Relaxation_rates
http://wiki.nmr-relax.com/Category:Model_free
http://wiki.nmr-relax.com/Category:Jw_mapping
http://wiki.nmr-relax.com/Category:Consistency_testing
http://wiki.nmr-relax.com/Category:N-state_analysis
http://wiki.nmr-relax.com/Category:Ensemble_analysis
http://wiki.nmr-relax.com/Category:Frame_order
http://wiki.nmr-relax.com/Category:Relaxation_dispersion
http://wiki.nmr-relax.com/Category:Relaxation

These categories will be much more stable than any pages on the wiki.  Each
category could then link back to the http://www.nmr-relax.com analysis page as
well to help with the search engine indexing.
