This would be the best.  Maybe just mail-to:relax-users as a default,
and that will do.  I can redirect development related posts to the
relax-devel mailing list later if necessary.  Remember that the user
will see zero difference between the GUI and the core of relax - to
them it is 100% the same thing.



For the bugs, we could make a link to the bug index?


This can be done, but this will be painful!  GUI bugs will be assigned
to you, and if there are 100 users of the GUI, you might get 100
separate bug reports for the same problem.  So to avoid this, you tell
the user to check that the bug has not already been submitted (plus
any other info needed).  In most cases, there will be a bug report on
the same thing, so this will automatically take care of most user's
problems.

There are 2 ways to do this:

1)  A new dialog in the GUI which tells the user this, and then opens
in a web-browser to https://gna.org/bugs/?group=relax (not the submit
form!) after they click on 'OK'.

2)  Instantly open a web page in a web-browser which tells the user
this, and then provide a link there to
https://gna.org/bugs/?group=relax.

What do you think?
