Posted by Edward d Auvergne on December 10, 2006 - 04:06:

Follow-up Comment #1, bug #7979 (project relax):

The issue is that the automatic updating of diffusion tensor data structures
in 'self.relax.data.diff' occurs through the 'self.__setattr__()' function. 
Therefore when MC simulation data is appended, this function is not called to
update the structures.  The append() function call needs to be caught so that
the diffusion tensor objects can be updated.
