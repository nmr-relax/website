Posted by Edward d'Auvergne on December 20, 2007 - 12:00:
Oh, note that these jw_mapping.py files have only partially been
migrated to the new design.  They still need a lot of work to be fully
functional again.  For example with the new molecule-residue-spin
structure, the reference to 'relax_data_store.res' will raise an
error.  Also for things like looping over residues, this is now
implemented in the generic_fns.selection.spin_loop() function which
should be imported.  It will be very useful to implement unit tests to
create a comprehensive net to catch all these problems.  And I am also
in the process of considering whether to split the
generic_fns.selection module into a few different modules, in which
case unit tests will again be very helpful for quickly resolving any
problems with this code refactorisation.

Regards,

Edward
