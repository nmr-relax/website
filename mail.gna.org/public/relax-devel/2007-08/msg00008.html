Follow-up Comment #2:

This is a bug in Numeric 24.2 and Python 2.5 (maybe only on 64-bit Linux
computers?).  This cannot be fixed in relax!  The bug does not exist if numpy
is used instead of Numeric.

Two options if you have this problem are to install Python 2.4 into
/usr/local/ and type 'python2.4 relax' to make relax use this version (or
changing the first line in the file 'relax' in the basedirectory is also an
option).  Otherwise older or future versions of Numeric may not suffer from
this issue.
