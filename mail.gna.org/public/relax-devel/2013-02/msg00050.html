Hi James,

I have now resolved this RelaxFault issue (the details of the fix can
be seen in the bug report https://gna.org/bugs/?20531).  This fix will
be present in the next version of relax, likely to be 2.2.3.  To
receive notice of the release, please see
http://marc.info/?l=relax-users&m=136087902719168&w=2.  However it
might take me a few weeks to release a new version.  If you cannot
wait, you could try using the subversion program to access the relax
repository where these changes have been made (for details see
http://www.nmr-relax.com/manual/Latest_sources_relax_repositories.html
or http://www.nmr-relax.com/download.html#Source_code_repository).

Cheers,

Edward
