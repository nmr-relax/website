-<H2><A NAME="SECTION081244000000000000000"></A> <A 
NAME="uf:_frame_order.average_position"></A>
-<BR>
-frame_order.average_position
-</H2>
-<IMG
-  WIDTH="128" HEIGHT="128" ALIGN="BOTTOM" BORDER="0"
- SRC="./frame_order.png"
- ALT="Image frame_order"> 
-
-<P>
-
-<H3><A NAME="SECTION081244100000000000000">
-Synopsis</A>
-</H3>
-
-<P>
-Define the mechanics of the average domain position.
-
-<P>
-
-<H3><A NAME="SECTION081244200000000000000">
-Defaults</A>
-</H3>
-
-<P>
-<DIV ALIGN="LEFT">
-<SPAN  CLASS="textsf"><SPAN  
CLASS="textbf">frame_order.average_position</SPAN>(pivot=`com', 
translation=False)</SPAN>
-
-</DIV>
-
-<P>
-
-<H3><A NAME="SECTION081244300000000000000">
-Keyword arguments</A>
-</H3>
-
-<P>
- <SPAN  CLASS="textsf">pivot:</SPAN>  The type of pivot to use for the 
rotation<A NAME="32487"></A> to the average domain position.  This can be one 
of `<TT>com</TT>' or `<TT>motional</TT>'.
-
-<P>
- <SPAN  CLASS="textsf">translation:</SPAN>  A flag specifying if the average 
domain position should be allowed to translate during optimisation.  If 
False, then translation can be disabled.
-
-<P>
-
-<H3><A NAME="SECTION081244400000000000000">
-Description</A>
-</H3>
-
-<P>
-Prior to optimisation, the mechanics of the average moving domain position 
must be specified.  Having the domain shifted to the correct average position 
is essential for understanding the dynamics as this information is the major 
contributor to the RDC and PCS.  The motional eigenframe (spherical or 
Euler<A NAME="32493"></A> angles)<A NAME="32494"></A> and ordering (via 
order<A NAME="32495"></A> parameters, cone angles<A NAME="32496"></A> or 
torsion angles)<A NAME="32497"></A> come second, and are therefore severely 
distorted by an incorrect average domain position.
-
-<P>
-There are two pieces of information affecting this average position - a 
rotation<A NAME="32498"></A> and translation.  For the rotation,<A 
NAME="32499"></A> a pivot point is required.  Note that this pivot is not 
related to the pivot of the motions.  However if you believe that your 
starting structure lies within the uniform distribution of positions of the 
domain motions, then the two pivots can be linked by setting the average 
position pivot to the motional pivot.  The default however is to set the 
pivot to the centre of mass (CoM) of the moving domain.
-
-<P>
-The second option allows the average domain position to translate during 
optimisation.  By default, only a rotation<A NAME="32500"></A> of the initial 
structure of the domain is rotated to the average position.  But if the 
rotation<A NAME="32501"></A> is not sufficient to shift the domain to the 
average position, then a translation will be required.  This option will be 
ignored if no PDC data is present, as RDCs do not contain information about 
the translation of the domain.
-
-<P>
-
-<H3><A NAME="SECTION081244500000000000000">
-Prompt examples</A>
-</H3>
-
-<P>
-To use the centre of mass as the rotational<A NAME="32503"></A> pivot and to 
allow translation of the average domain position during optimisation, type 
one of:
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate('com', True)
-</PRE>
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate(translation=True)
-</PRE>
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate(pivot='com', translation=True)
-</PRE>
-
-<P>
-To use the motional pivot as the average domain rotational<A 
NAME="32510"></A> pivot while disallowing translation of, type one of:
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate('motional')
-</PRE>
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate('motional', False)
-</PRE>
-
-<P>
-<PRE>[numbers=none]
-relax&gt; frame_order.translate(pivot='motional', translation=False)
-</PRE>
-
-<P>

-<H3><A NAME="SECTION081245100000000000000">
-Synopsis</A>
-</H3>
-
-<P>
-Set the number of integration points used in the quasi-random Sobol' 
sequence<A NAME="32525"></A> during optimisation.
-
-<P>
-
-<H3><A NAME="SECTION081245200000000000000">
-Defaults</A>
-</H3>
-
-<P>
-<DIV ALIGN="LEFT">
-<SPAN  CLASS="textsf"><SPAN  
CLASS="textbf">frame_order.num_int_pts</SPAN>(num=200000)</SPAN>
-
-</DIV>
-
-<P>
-
-<H3><A NAME="SECTION081245300000000000000">
-Keyword arguments</A>
-</H3>
-
-<P>
- <SPAN  CLASS="textsf">num:</SPAN>  The number of integration points to use 
in the Sobol' sequence<A NAME="32533"></A> during optimisation.
-
-<P>
-
-<H3><A NAME="SECTION081245400000000000000">
-Description</A>
-</H3>
-
-<P>
-This allows the number of integration points used during the Frame Order 
target<A NAME="32535"></A> function optimisation to be changed from the 
default.  This is used in the quasi-random Sobol' sequence<A 
NAME="32536"></A> for the numerical integration.
