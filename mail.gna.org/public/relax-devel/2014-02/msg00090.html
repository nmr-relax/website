Posted by Edward d'Auvergne on February 17, 2014 - 17:41:
Hi Troels,

This looks like a problem with the current data checks prior to
executing the auto-analysis.  To fix this, I would recommend the
following steps:

1)  First, I would recommend creating a bug report.  It would be good
to attach a relax save state for this problem with data from a single
spin system, saved just before clicking on 'Execute'.  If you have
such a state file already for the entire system, the spin editor
window can be used to delete all but one spin (which may itself reveal
more bugs, but this is a good thing :).

2)  Once the bug report is created with the save state attached, a
system test can be created which loads the state file, saved in the
test_suite/shared_data directories, and executes the auto-analysis,
triggering the bug.

3)  Fix the bug and close the report.

4)  Optimisation of the system test - by making sure it doesn't take
more than 1-2 seconds.  Hidden optimisation settings in the
auto-analysis can be used for this.

Would you like to give this a go?

Cheers,

Edward
