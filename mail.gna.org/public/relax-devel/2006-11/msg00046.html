Gary,

Would you like to manage the 'test_suite' branch during its,
hopefully, short lifetime?  This involves porting all revisions
applied to the 1.3 line (e.g. Chris' r2719).  This can be done in
bulk, i.e. port multiple revisions into one commit.  To manage a
branch you need to keep track of the revisions in the parent branch,
and no revision must be ported twice to the branch (otherwise there'll
be a mess to sort out).  This will probably involve porting the
changes to the 'tensor_pdb' branch which will appear as one massive
revision within the 1.3 line one I merge it back in.

The other components to managing the branch would be to direct its
development through this list, and finally merge it back into the 1.3
line once the unit test and the full test suite is in a descent state.

Edward
