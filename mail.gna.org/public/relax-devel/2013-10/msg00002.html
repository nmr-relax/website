Follow-up Comment #1:

This is a deliberate design decision for the GUI, introduced by Michael Bieri.
 If the user does not want anything set for an argument, in the GUI this is
represented by an empty field rather than a field containing the text "None". 
A GUI user does not need to know Python programming concepts such as the
special 'None' Python type (or 'True', 'False', and other special Python
objects).  They will see it in the log messages, but that is not important.  I
am closing this report as 'Wont Fix' as this is part of the design.
