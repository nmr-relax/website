Posted by Edward d'Auvergne on May 15, 2013 - 11:43:
Hi,

Thanks for these reports.  The print statement causing the error you
see is clearly very wrong!  It should be changed from:

print("A:\n" % cdp.align_tensors[0])

to:

print("A:\n%s" % cdp.align_tensors[0].A)

But what is more worrying is that this has not caused an error on any
of my test machines.  This includes GNU/Linux with Python 2.3, 2.4,
2.5, 2.6, 2.7, 3.1, 3.2, and 3.3 (I have all of these versions self
compiled with all packages required just to test how relax behaves for
different users, and even versions back to 1.0); Mac OS X with the
relax DMG distribution, Fink, and source; and MS Windows 2000 and 7
with various Python versions.  This print statement is quite old, and
on none of those systems has the error occurred.  And no user has
reported it.  But it is clearly an error.  It seems like it is only
triggered by WinPython!

The cdp if the current data pipe object in the relax data store.  Most
of the complex objects in the data store are custom Python objects.  I
have been testing with the code:

class X(dict):
    pass

x = X()
print("x: " % x)

If this is placed into a text file and run by all Python versions >=
2.2 on my GNU/Linux system, no error occurs!  With the code:

x = {'a': 1}
print("x: " % x)

I can see that the behaviour of accepting this print statement error
was introduced between in the very early days of Python.  In Python
1.2 I see:

x:

Whereas in Python 1.1 I see:

Traceback (innermost last):
  File "b", line 2, in ?
    print("x: " % x)
TypeError: not all arguments converted

This is part of the Python interpreter, so there is nothing I can do.
I don't know why WinPython functions differently.  Therefore I think I
will just have to fix this problem and rely on you to report (and
maybe fix) any other basic issues that may appear in the future.  I
hope there are no other corner cases.  I'm sure if others try
different Python interpreters, such as PyPy, in the future other
issues will be uncovered.

Regards,

Edward
