Posted by Edward d'Auvergne on December 08, 2014 - 09:17:
Hi,

The wizard graphics in the graphics/wizards/ directory are not square,
but all icons in graphics/oxygen_icons/ and graphics/relax_icons/ are.
Those directories are to separate the purpose of the icons from the
other graphics, as well as for respecting the different licences.
However the square icon could serve as a base for non-square wizard
graphics.  Actually, I have been thinking about expanding the wizard
graphics by superimposing, for example, the file-open, file-save etc.
oxygen icons in the bottom right hand corner of each of the wizard
graphics, for having a slightly different graphic per user function.
Anyway, in this case, there is an overlapping purpose as you are using
a custom size icon for the wizard graphic.  However if your wizard
graphic were to be expanded in height, it would need to be shifted to
graphics/wizards/ to fit, as it will no longer be an icon, which are
square.

There is one more thing for the icon graphics, which I'll look at
fixing.  For the 128x128 versions of the icons, a Gzipped EPS graphic
is always required for the compilation of the user manual.  The
automated compilation expects the file to be present.  I normally
create this with the Gimp, but the PS bounding box is different in
later versions so I might have to manually tweak this.

Regards,

Edward
