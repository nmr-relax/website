Details:

When making a graph of R1_rho as function of theta, the current setup in relax
is by interpolating R1_rho points by extending the dispersion points
(spin-lock field strength [w1]), and back-calculate the the R1_rho via the
used model function.

This can lead to some spurious graphs, which have been attached.
ramp_w1_ramping_spin-lock_field_strength.png

The contribution from Rex: sin^2(t)*(phi_ex*kex/(kex^2+w_e^2)) when ramping
w_1, leads to some "shoulders", which can lead to a conclusion of wrong
input.

Simpler graphs would be expected, if ramping spin-lock offset [Omega]
instead.

See attached 
ramp_Omega_spin-lock_offset.png

The graphs are also available at:
http://wiki.nmr-relax.com/DPL94_math

The code that needs to be modified resides in:
specific_analyses.relax_disp.optimisation.optimisation.back_calc_r2eff(spin=cdp.fakespin,
spin_id=fake_spin_inte, spin_lock_nu1=spin_lock_nu1)

# Which calls
specific_analyses.relax_disp.disp_data.return_offset_data()
chemical_shifts, offsets, tilt_angles, Delta_omega, w_eff =
return_offset_data(spins=[spin], spin_ids=[spin_id], field_count=field_count,
fields=spin_lock_nu1)

These should be modified to accept interpolated offset data.
