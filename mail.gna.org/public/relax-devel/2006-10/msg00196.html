Details:

The problem is that for the 1.2.5 release onwards (see the CHANGES file), the
local tm parameter was renamed from 'tm' to 'local_tm'.    Therefore the
'params' column in the model-free results file when using the local tm models
uses different names after the 1.2.5 release.  When trying to read a results
file from 1.2.4 relax versions or earlier, a RelaxFault is encountered.  The
resulting messages make no sense and do not indicate what the issue is.
