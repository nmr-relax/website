Details:

In Mac OS X ("Lion", 10.7.3), the self-contained relax Application bundle is
crashing and is giving just an "relax error" and the option to "terminate" or
"open the Console". 

In Console.app I find the error messages wich can be found in the attached
relax-launcher-console.log

Interestingly, relax runs just fine if I run the relax launcher (?) binary in
"Contents/MacOS":

$ cd /Applications/relax.app/Contents/MacOS/
$ ./relax 

So obviously the wrapper for the Mac OS bundle is failing somehow, as the
program itself is actually working perfectly fine.

*** 

Trying to run relax GUI using the system-installed packages results in an
error. This may be due to errors in my installs of python, wxPython, numpy or
whatever.

$ ./relax --info --tee info_log
$ ./relax --gui --tee gui_log

See the files relax-info.log and relax-gui-info.log for details.
