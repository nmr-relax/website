Posted by Edward d'Auvergne on March 16, 2006 - 07:11:
To try to fix the memory leak problem, I've added Py_DECREF() calls to
the 'func' and 'dfunc' functions to deallocate the memory used by the
parameter vector 'numpy_params'.  This function is part of the
Python/C API.  I've also added declarations for the pointer '*params'
used within those functions.  The main reason I made the pointers
'*values, *sd, *relax_times, *scaling_matrix' external was to make the
data persistent so that the values are set once during the setup and
then each 'func' call uses that data.  As for the layout of the
relax_fit.c code, I've simply followed the instructions in the
Python/C API found at http://docs.python.org/api/api.html.  I've
separated the chi2.c function, etc, so that these can be reused by
future C modules.  I can't currently test if the memory leak has been
fixed as the changes to the residue selection code is causing
problems.

Thank again for the help,

Edward
