Posted by Edward d'Auvergne on April 03, 2008 - 11:02:
Please note that the changes I have made here may not yet be the final
form.  It could be unstable until I have something fully functional
implemented.  The creation of a mapping between the structural code
(PDB readers, etc.) and the relax data store is still very much up in
the air.  I would like to have this evolve into a stable API in which
the two talk to each other independent of their implementation
details, so that other PDB or other structural file readers can be
added without needing to modify any other parts of relax.

Regards,

Edward
