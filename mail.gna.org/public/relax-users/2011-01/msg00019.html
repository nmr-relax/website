Posted by Edward d'Auvergne on January 11, 2011 - 14:45:
Dear all,

If you have been using the very recent relax version 1.3.6 (or the 1.3
main line or branches from the relax repository) you may have been
affected by a recent bug in relax (found by Vitaly Vostrikov and
detailed at https://gna.org/bugs/?17458).  This would only be a
problem if relax was used for model-free analysis.  If you have used
the dauvergne_protocol.py or full_analysis.py script for the analysis,
there is a simple check to see whether your results have been
influenced.  Open the file 'prolate/init/results.bz2' and search for
the 'spheroid_type' XML entry.  If you see:

            <spheroid_type>
                'oblate'
            </spheroid_type>

in this file, then your results are not correct and you will need to
re-run the prolate spheroid diffusion tensor global model.  This entry
should say 'prolate'.  As this is a very recent bug, most results
should hopefully not be affected.  Sorry if this has caused any
inconveniences.

Sincerely,

Edward
