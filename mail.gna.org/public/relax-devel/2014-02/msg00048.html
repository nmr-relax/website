Follow-up Comment #1:

I've tested the problem and can confirm that this is a bug.  Using the PDB
file attached to sr #3110
(https://gna.org/support/download.php?file_id=19990), the bug can be
reproduced in relax with the two simple user function calls of:

pipe.create('temp', 'mf')
structure.read_pdb(file='BpUreE_apo_model_full.pdb', set_mol_name='SpUreE_SC')


However as this is a support request rather than bug report, this will be
closed as 'invalid'.  Please submit this as a bug report using the link
https://gna.org/bugs/?func=additem&group=relax.  Please also attach the PDB
file to that bug report and include the relax output text shown below.  Once a
bug report is submitted, I'll create a system test in relax (including a
truncated version of the PDB file) to catch it.  This will allow me to
eliminate the bug within a few minutes.  Cheers!
