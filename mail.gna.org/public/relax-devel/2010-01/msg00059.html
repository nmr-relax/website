I'll revert this, it was just a test.  As it's in one commit, this is
very easy to do.  I think in the end it would be better to change the
'+' button because it isn't that obvious that it opens a file opening
dialog.  But this would require a slight layout redesign - which we
should consider anyway to allow 4 field strength data, or 5, or even
cross-correlated relaxation to be supported.



What do you think about some more modern symbols,, such as provided in
many 'Vista icon patches' ? I am not a big fan of oxygen symbols, as
they look ugly....


Unfortunately, using these and many other icons set is totally
illegal.  We have to be able to distribute these, and the licence must
be compatible with the GPL so that downstream users can legally
distribute relax, parts of relax, or modifications of relax.  My
reasoning for using the Oxygen set (http://www.oxygen-icons.org/) is
that is more aligned with the theme colour of relax - blue
(http://www.nmr-relax.com/).  It is a very blue icon set, that's
professionally made.  The icons are are also licenced under the LGPL
(http://www.gnu.org/copyleft/lesser.html) which is perfect for the GPL
(http://www.gnu.org/licenses/gpl.html).



I will sort out some alternatives and upload them to the task so you can
make up your opinion. But it is certainly a nice cosmetic enhancement!


They do look nice, but we have to remain legal.  As for the icons
added, the more pictures in a GUI the better!  It's cosmetic, but that
is what is important in a GUI.
