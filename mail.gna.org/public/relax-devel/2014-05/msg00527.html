Hi Troels,

I would recommend setting the dim argument to (None, 3).  The current
setting means that there will be three 3D points, and this can never
be changed.  By setting the first value to None, that will mean that
the user can change the number of points.  If this causes errors,
please just file a bug report.  I now know how to create a GUI test
for the user function window and then I can catch and fix it.

Cheers,

Edward
