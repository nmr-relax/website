Posted by Edward d'Auvergne on June 20, 2013 - 18:06:
Hi Troels,

This is actually quite a common problem on MS Windows with certain
wxPython versions that always causes a lot of pain to try to fix.
Many times it is due to a bug in wxPython itself and I have to make
relax avoid that bug.  Does it also fail if you turn debugging on?
Does it fail if you run the GUI tests through the GUI?  Which Python
system is this running on?  Could you also run:

$ relax -i --log relax_info.txt

and copy and paste the contents of the relax_info.txt file?  The fix
might be as simple as switching to wxPython 2.9 if you are using 2.8,
or switching to 2.8 if you are using the 2.9 development branch.

Regards,

Edward
