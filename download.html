Current releases

General information

The current development release is relax version 1.3.3.

The current stable release is relax version 1.2.15.

For most analysis types, relaxation curve fitting, model-free analysis, etc., the development release is adequate and recommended. If there are problems, a bug report can be lodged or the stable releases used.

relax binaries have been pre-compiled for a number of computer architectures. These are based on Python 2.5 but can work with Python 2.4 on certain operating systems and certain architectures. However if not, or if binaries for your system do not exist, please download the source release (the compilation of binaries is not a pre-requesite for using relax). Note that some binary distribution files may not correspond to the most up to date version of relax. If so, please download the up-to-date source release instead. If you would like to contribute binary packages for your specific architecture, please read the 'Open source infrastructure' chapter of the relax manual for details.

For details about the relax GPG signatures and their usage, see the Security section at the end of this page.

Dependencies

If they exist, please install your distribution specific packages for the following dependencies.

  • Python >= 2.4: Python-2.4.3.tar.bz2
  • Scientific Python (optional) >= 2.8: ScientificPython-2.8.tar.gz

For the 1.3 releases, the following dependencies are also needed.

  • minfx >= 1.0.1: minfx-1.0.1.tar.bz2
  • numpy >= 1.0.4: numpy downloads

Pre-compiled binary releases of certain relax versions are available for Mac OS X within the Fink project. These can be installed for Python 2.4 with the command:

> fink install relax-py24

or for Python 2.5 with:

> fink install relax-py25

The relax releases packaged within Fink can been browsed here. If the desired version is not available, please download the relevent source package below.

Compilation

For compiling the C modules relax uses the SCons cross platform compilation tool. Building these modules is currently only required for the exponential curve fitting used to determine the R1 and R2 relaxation rates. By downloading and installing the program and running the command:

> scons

in the relax base directory, the modules should automatically compile. This should work on all operating systems assuming that a C compiler is present. If an error occurs on MS Windows, read the instructions here.

Development platforms

The development platform is required for compiling the C modules through SCons, for compiling the LaTeX manual, and generating relax distribution packages.

Security

To check that the file hasn't been compromised, firstly download the GPG signature for the distribution package. Then to test this signature, download the relax public key and run the command:

> gpg --verify --default-key relax relax-x.x.x.release_type.file_type.sig

where x.x.x is the program version; release_type is one of 'GNU-Linux.i686', 'Win32', or 'src'; and file_type is either 'tar.bz2' or 'zip'. The GPG program can be downloaded from the GNU Privacy Guard web site.
