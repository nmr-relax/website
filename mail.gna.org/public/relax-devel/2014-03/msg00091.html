What would you propose as an implementation?  One idea might be to add
a new page to the new analysis wizard in the GUI which appears if the
dauvergne_protocol auto_analysis is selected which explains what that

The mailing list has shown that last time period, that between 2-3 users
complain about the execution of the dauvergne protocol.

This seems to stem from the lack of recording of two fields.

It should be very very clearly stated in the GUI, and in manual that data 
for
at least two fields is necessary.

The warning message is not clear enough:
Insufficient relaxation data, 4 or more data sets are essential for the
execution of the dauvergne_protocol auto_analysis.

There should probably be added that at least two fields are necessary.
