Posted by Edward d'Auvergne on February 10, 2014 - 12:07:
Dear Stefano,

Would you be able to copy and paste the full text from the relax
controller window?  It is best not to attach files to public mailing
lists as this causes undue strain on the open source infrastructure.
For example the messages will no longer be archived at
http://www.mail-archive.com/relax-devel@xxxxxxx/.  And the image will
be amplified and sent out to everyone subscribed to the mailing list
and also be archived at
https://mail.gna.org/public/relax-devel/2014-02/msg00029.html and
http://thread.gmane.org/gmane.science.nmr.relax.devel/4966/focus=4970.

For the warnings, this is difficult to tell as it cannot be seen what
user function was executed to obtain these.  They seem to be harmless
nevertheless.  The last text about the data loaded into the relax data
store seems to show that the data was successfully loaded.  Note that
due to a bug in certain wxPython versions (http://wxpython.org/), the
progress bars on Mac OS X systems will appear to always be active,
once they have been 'turned on' - this is not fixable within relax
(the GUI in relax is implemented in wxPython).  So on Mac OS X
systems, this can be ignored.

Regards,

Edward
