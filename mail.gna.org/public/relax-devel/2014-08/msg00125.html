Hi Troels,

I don't agree on this order:

    expected_result = [MODEL_R2EFF, MODEL_NOREX_R1RHO_FIT_R1,
MODEL_MP05_FIT_R1, MODEL_TAP03_FIT_R1, MODEL_TP02_FIT_R1,
MODEL_DPL94_FIT_R1, MODEL_NS_R1RHO_2SITE_FIT_R1]

as the TP02 model should be before MP05.  The TP02 parameters could be
used as the starting point for the MP05 model (though I doubt anyone
will optimise both of these models at the same time, it might happen).
MP05 should be before the numeric model, as it is.  And the rest I
would not use nesting for, so the order is unimportant.  Though I
don't see why DPL94 comes after all of the Palmer models.

Regards,
