Hi,

I would recommend just adding the peak lists for the R1rho dispersion
data to start with the first commit (and maybe the peak lists for the
R1 data as well).  I would also just have the rx.out files copied to
the base directory as, for example, 'R1.500.out'.  I am guessing that
these are R1 standard rates.  There is no need for the relax results
files and grace plots for these.  If you wish, you could include the
peak lists for the R1 data, but the rest is not necessary for the
purpose of this data and just takes up large amounts of space.

Another suggestion would be to not use the ending '*.ser' for the peak
lists.  This is confusing as it then looks like Bruker 'ser' files -
i.e. the FID files.  Is this file ending automatically generated by
some NMR software?  Anyway, for a first commit, just having the text
files would be best.  The final relax results and grace files can be
added once the analysis is debugged and functional, and these could be
simply for a small spin subset.  These are large files which will
change later anyway.  They are in the other dispersion data
directories so that relax users can copy the data and see if they can
produce the same results - strangely some users seem to need this.

The next step will be for you to make a request to join the relax
project at Gna! ;)

Regards,

Edward
