Follow-up Comment #2, bug #21814 (project relax):

This problem is due to a PDB record violation created by the MODELLER
software.  According to the PDB definition, every PDB line/record must be 80
characters.  All files downloaded from the PDB will have all records padded 
to
80 characters with whitespace.  For many records in the attached file, this 
is
not the case.

It should be easy for relax to handle the MODELLER corruption.  Each record
can simply be padded to 80 characters with spaces - as a PDB file should be 
-
prior to trying to extract the information from the record.
