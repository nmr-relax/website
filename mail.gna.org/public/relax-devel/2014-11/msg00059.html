I found the nmrglue mailing list, though it is instead a Google forum

http://news.gmane.org/gmane.science.nmr.relax.scm,

archives, see http://www.nmr-relax.com/communication.html), as well as
in the relax API documentation (http://www.nmr-relax.com/api/).  This
should massively boost the search engine ranking for nmrglue!

On 27 November 2014 at 13:49, Edward d'Auvergne <edward@xxxxxxxxxxxxx> wrote:

Hi Troels,

I think nmrglue is too large to include like this within the relax
source directories.  I think this commit should be reverted using the
standard subversion command:

$ svn merge -r26778:26777 .

including this command in the commit message.   Note that you should
probably also ask on the nmrglue mailing lists first if they are ok
with this (though I don't know if they have a public mailing list
setup, or at least I cannot find one at the moment).  There should be
an introduction message to their mailing lists anyway, with lots of
links to http://www.nmr-relax.com (to introduce relax and your idea),
the task, and the relevant mailing list messages.  Because of the
licensing, we are legally allowed to do this, but it is still nice to
ask.

Anyway, we actually only require the nmrglue directory of the nmrglue
distribution for including with relax.  As this is designed as a
standard Python package, this is how it is done.  For example from the
docs/devel/Release_Checklist document, we do exactly this for minfx
and bmrblib.  I bundle these packages only when I create the relax
distribution files for release using the commands:

$ svn co svn+ssh://bugman@xxxxxxxxxxx/svn/minfx/tags/1.0.11/minfx
$ svn co svn+ssh://bugman@xxxxxxxxxxx/svn/bmrblib/tags/1.0.3/bmrblib

This only includes the 'minfx' and 'bmrblib' directories and ignores
the rest (http://svn.gna.org/viewcvs/minfx/tags/1.0.11/).  We could do
the same for nmrglue here.  Simply add the git command required to the
docs/devel/Release_Checklist document for doing this.  Maybe we could
do a sparse checkout.  Though as this is rather complicated and I need
to perform this for 32 and 64-bit GNU/Linux, the source distributions,
Mac OS X dmg file, and MS Windows zip file.  So maybe we could just
add this directory to the relax sources simply as extern/nmrglue.  For
version 0.4, this directory is only 684K.  That would be a reasonable
size to include.  The README file could include instructions for
curious users to obtain the rest of the nmrglue release.

Cheers,

Edward
