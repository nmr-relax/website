This is a good idea.  Note that setting the MC sim number to -1 is
only possible in a script, and not the GUI.  Oh, you changed that
later.  An alternative would be to add the curve_fit_errors argument
which defaults to 'MC' but can be set to 'covar'.  I would suggest
also having a RelaxWarning in the auto-analysis about the covariance
matrix being of lower accuracy than MC sims, just so the user is more
aware.
